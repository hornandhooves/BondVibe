# KQA-001 — Saludo "Good morning" en la madrugada

- **Severidad:** P4 (cosmético / pulido)
- **Estado:** En desarrollo (decisión tomada)
- **Pantalla / Flujo:** Home (HomeScreen) — saludo del encabezado
- **Entorno:** iOS Simulator (iPhone 17 Pro, iOS 26.1)
- **Fecha:** 2026-07-17

## Resumen
Entre 00:00–04:59 el Home saluda "Good morning" porque `getGreeting` usa `hour < 12 → morning`.

## Reproducir
1. Hora del dispositivo entre 00:00 y 04:59 (visto a las 2:06 a.m.).
2. Abrir app → Home → muestra "Good morning".

## Causa (código)
`src/screens/HomeScreen.js` (~L80–84):

    const hour = new Date().getHours();
    if (hour < 12) return t("home.greetingMorning");
    if (hour < 18) return t("home.greetingAfternoon");
    return t("home.greetingEvening");

## DECISIÓN (2026-07-17) — opción 1A
Añadir saludo de **noche** con **key i18n nueva** (sin hardcode). Franjas:
- 05–11 → `home.greetingMorning`
- 12–17 → `home.greetingAfternoon`
- 18–23 y 00–04 → `home.greetingNight`  (NUEVA)

Copy EN sugerido: "Good evening" (saludo natural) · ES: "Buenas noches". Ajustable por Design.

## Fix (dev, en VSCode)
1. Extraer a util puro `src/utils/greeting.js`:

       export function getGreetingKey(hour) {
         if (hour >= 5 && hour < 12) return "home.greetingMorning";
         if (hour >= 12 && hour < 18) return "home.greetingAfternoon";
         return "home.greetingNight";
       }

2. En HomeScreen: `t(getGreetingKey(new Date().getHours()))`.
3. Añadir `home.greetingNight` en EN y ES (lo valida `scripts/check-i18n-parity.js`).

## Test de regresión (listo para agregar tras el fix)
