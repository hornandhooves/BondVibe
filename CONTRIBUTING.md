# Contributing to Kinlo

## Branch Strategy

- `main` - Production
- `develop` - Integration
- `feature/BV-XXX-description` - Features

## Commit Format
```
feat(BV-XXX): description
fix(BV-XXX): description
docs: description
```

## Workflow

1. Create branch from `develop`
2. Make changes
3. Commit with conventional format
4. Push and create PR
5. Merge after review
